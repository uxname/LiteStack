## Краткие выводы

- REST-поверхность узкая (`/health`, `/upload`, `/uploads/*` + dev-страницы), документация openapi.yaml синхронизирована с реальными маршрутами автоматическим тестом `TestOpenAPISpecMatchesRoutes` (двусторонняя проверка: каждый маршрут задокументирован, каждая запись спеки имеет маршрут) — рассинхрона нет.
- HTTP-коды и именование полей корректны и консистентны.
- Главная проблема — **утечка внутренних деталей ошибок через GraphQL** (API-05 / ERR-02): отсутствует production-маскировка, сырые ошибки БД доходят до клиента.
- Конвенции ошибок REST и GraphQL расходятся (REST без строкового `code`/`requestId`) — машиночитаемость REST ниже (API-03).
