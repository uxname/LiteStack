# Audit Report: Error Handling & Resiliency — 2026-06-20 21:47

Runtime: **Go 1.26** (модуль `github.com/uxname/liteend-go`). Чеклист изначально заточен под Node.js — JS-специфичные пункты переведены в эквиваленты Go (`context.Context`, `signal.NotifyContext`, `recover()`). Baseline пуст — все находки активны.

В целом обработка ошибок в проекте сделана зрело: ошибки почти везде оборачиваются через `%w`, есть явные таймауты на БД/Redis/OIDC, реализован graceful shutdown через `signal.NotifyContext` + `srv.Shutdown`, паники перехватываются в HTTP-, очереди- и подписочных слоях. Основные проблемы — утечка внутренних деталей ошибок наружу (в GraphQL-ответы и в публичный `/health`).

## Заметки по graceful degradation (вне FAIL)

- Rate-limit **fail-open** при недоступности Redis (`ratelimit.go:31`) — осознанный выбор «доступность важнее троттлинга», задокументирован в коде. Это приемлемо для rate-limit, но стоит помнить: при падении Redis защита от перебора отключается.
- Backup использует `mu.TryLock()` и пропускает запуск, если предыдущий ещё идёт (`backup.go:62`) — корректная защита от наложения.
- Auth-middleware при ошибке верификации токена возвращает `nil` (анонимный доступ), не роняя запрос (`auth/middleware.go:95-104`) — корректно для `Optional`; enforcement делают guard'ы резолверов (`auth.Require`).
