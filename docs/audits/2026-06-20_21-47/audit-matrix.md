# Матрица — LiteEnd-Go (backend) — 2026-06-20

## TL;DR
- **Архитектура:** монолит на Go (один процесс — HTTP + GraphQL + REST + фоновые воркеры), плюс соседние контейнеры (Postgres, Redis, бэкап, дашборды) в одном docker-compose.
- **Компонентов:** 9
- **Критических рисков (🔴/🟠):** 6
- **Главный риск:** при старте приложение жёстко требует и Postgres, и Redis. Если хоть один недоступен в момент запуска — процесс падает и не поднимается (`internal/app/app.go:48-83`). Под нагрузкой во время работы тяжёлый SQL-запрос может занять все 10 соединений пула (нет таймаута на запрос) и подвесить весь сайт.

Процесс не поднимется, Docker будет рестартить по кругу | 🟠 | Сознательный fail-fast. Healthcheck и `restart: unless-stopped` это страхуют. ## Postgres

**Роль:** основное хранилище (профили, метаданные загрузок). Доступ через пул pgx + sqlc. • **Где:** Docker (`db`), volume `postgres` • **Найден:** `internal/db/pool.go:22`

| # | Сценарий | В чём опасность | Риск | Решение | Файл |
|---|----------|-----------------|------|---------|------|
| A1 | Тяжёлый/зависший запрос | Нет таймаута на отдельный запрос. Один медленный SELECT держит соединение, при 10 таких пул исчерпан → весь сайт ждёт и отдаёт ошибки. БД при этом «жива» | 🔴 | Добавить `statement_timeout` (напр. 5s) в строку подключения или на пул, чтобы зависший запрос не съедал пул | `internal/db/pool.go:28` ❌ |
| A2 | БД недоступна при старте | Миграции ретраятся (5 попыток, backoff до 5s), потом ошибка и выход. Это норм для старта | 🟢 | Ретраи уже есть | `internal/db/migrate.go:32-52` ✅ |
| A3 | БД недоступна во время работы | `Ping` в `/health` вернёт error → healthcheck отдаст 503 → Docker пометит контейнер нездоровым. Запросы упадут, но не зависнут навсегда (есть `ConnectTimeout` 10s) | 🟠 | Уменьшить таймаут ожидания соединения под нагрузкой; см. A1 про statement_timeout | `internal/db/pool.go:30` |
| A4 | Гонка find-or-create профиля | Два параллельных логина одного `sub` → INSERT-конфликт ловится по коду 23505 и перечитывается. Дубля и 500 не будет | 🟢 | Защита уже есть | `internal/profile/service.go:87-101` ✅ |
| A5 | Пул всего 10 соединений | Под всплеском трафика 10 может не хватить даже на быстрых запросах → очередь ожидания. Для монолита с воркером в том же процессе маловато | 🟡 | Поднять `DBPoolMax` под реальную нагрузку и измерить (`config.DBPoolMax`) | `internal/config/constants.go:43` |

| # | Сценарий | В чём опасность | Риск | Решение | Файл |

Выглядит как массовый разлогин | 🟠 | Таймаут есть ✅. Различать «плохой токен» (401) и «провайдер недоступен» (503), чтобы клиент не считал, что токен протух | `internal/auth/middleware.go:95-98` ⚠ |
