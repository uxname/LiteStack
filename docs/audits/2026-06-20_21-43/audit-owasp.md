# Audit Report: OWASP Application Security — 2026-06-20 21:43

Проект — клиентский фронтенд (React 19 + TanStack Start SSR, OIDC через Logto,
GraphQL через urql). Своей базы данных и серверного хранилища паролей у него нет:
аутентификацию полностью ведёт внешний OIDC-провайдер (Logto), пароли там же.
Поэтому injection / IDOR / хранение паролей оцениваются применительно к реальному
коду фронтенда (SSR-обработчик, GraphQL-клиент, OIDC, загрузка аватара, инлайн-скрипт).

Runtime: Node.js / TypeScript. Baseline: пустой (`accepted: []`).

## Замечания (вне чеклиста)

- **Уязвимые зависимости (`npm audit`).** `undici` — **high** CVE (TLS cert validation bypass, HTTP header/response injection, cache poisoning и др.); `@opentelemetry/core` + транзитивные — 3× **moderate** (unbounded memory в W3C Baggage). `undici` тянется транзитивно (через Nitro/server-стек), на SSR-сервере он используется для исходящих запросов. Рекомендуется `npm audit fix` / поднять версии. Это самостоятельная зона, не входит в OWASP-чеклист.
- **Access-токен в `localStorage`** (`src/features/auth/api/oidc-client.ts:30`) — стандартный для SPA подход с `oidc-client-ts`, но при любом XSS токен читается напрямую. Жёсткий CSP (см. OWA-05) — основная компенсирующая мера; учтено в решении OWA-05.
- **Инлайн-скрипт темы** (`src/routes/__root.tsx:32-53`) безопасен: содержимое статическое, `JSON.parse` обёрнут в `try/catch`, парсится только blob из localStorage самого приложения, а присваивается лишь при `theme === 'dark' || 'cmyk'` (whitelist) — внедрение произвольного `data-theme` через подмену localStorage невозможно. Единственный нюанс — он требует послабления CSP (nonce/hash), см. OWA-05.
