# Матрица — LiteStack frontend — 2026-06-20

## TL;DR
- **Архитектура:** монолитный фронтенд (SPA + SSR на одном Nitro-сервере), Feature-Sliced Design. Внешние зависимости — OIDC-провайдер (Logto) и GraphQL/REST бэкенд.
- **Компонентов:** 8 (3 свои клиентские подсистемы + SSR-сервер + 2 свои инфраструктурные + 2 внешних сервиса).
- **Критических рисков (🔴/🟠):** 3 (2× 🟠, 1× условный 🟠 — отсутствие CSP).
- **Главный риск:** загрузка аватара (`fetch` на бэкенд) идёт без таймаута и без отмены — если бэкенд завис, форма профиля блокируется навсегда, пользователь не может ни сохранить, ни уйти из состояния загрузки.

| # | Сценарий | В чём опасность | Риск | Решение | Файл |

| A2 | Токен протух во время сессии | `automaticSilentRenew: true` сам молча обновляет токен; при смене токена urql-клиент пересоздаётся (`AppProviders.tsx:70`) и кэш сбрасывается → защита есть | 🟢 | Ничего делать не надо — silent renew + пересоздание клиента уже закрывают | `src/features/auth/api/oidc-client.ts:29` ✅ |

## 4. SSR-сервер

**Роль:** рендер HTML, резолв локали из cookie/Accept-Language, установка cookie локали • **Где:** Docker (Nitro-бандл) • **Найден:** `src/server.ts:17`

| # | Сценарий | В чём опасность | Риск | Решение | Файл |
|---|----------|-----------------|------|---------|------|
| D1 | SSR-рендер упал внутри `paraglideMiddleware` | `await handler(...)` без `try/catch` → при падении рендера ошибка уходит в Nitro без app-фолбэка, и `Set-Cookie` с локалью не выставится → первый визит может получить непредсказуемый ответ и потерять локаль | 🟡 | Обернуть `await handler(...)` в `try/catch`: логировать в Sentry и вернуть аккуратный 500 с сохранением `Set-Cookie` | `src/server.ts:23` ❌ |
| D2 | Падение/перезапуск процесса (SIGTERM, unhandled rejection) | В app-коде нет `process.on('SIGTERM'/'uncaughtException')`; обработка лежит в сгенерированном Nitro-бандле (`.output/`, вне исходников), статически не подтверждается. Сервер stateless (нет своего DB-пула) → graceful drain менее критичен | 🟡 | Проверить поведение Nitro-пресета `node-server` на SIGTERM; при необходимости добавить process-обработчики в кастомную точку входа | `src/server.ts` (⚠ не проверено — код в `.output/`) |
