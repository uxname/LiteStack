# ideas/

Free-form backlog of ideas for LiteStack and the full-stack product built on it.
Lower commitment than `tasks/` — capture first, refine later.

## How to use

- One file per idea: `ideas/<slug>.md` (e.g. `ideas/realtime-notifications.md`).
- No fixed format. A title, a paragraph of what/why, and rough scope is enough.
- When an idea is ready to act on, promote it into a `tasks/` spec (copy `tasks/TEMPLATE.md`)
  and link back here.
